# RxJS

Библиотека RxJS — библиотека для работы с асинхронными данными в стиле реактивного программирования через использование паттерна, который называется `Observable`

```js

// status
// data

// one-time thing
const myPromise = fetch('/api/users')
  .then((res) => {
    console.log(res);
  })

// Reactive Programming
const myObservable = new Observable()
  .subscribe((res) => {
    console.log(res);
  });
```

Отличие (и основа реактивности) библиотеки RxJS от Promise заключается в том, что `Observables` которые создаются библиотекой могут запускать множественные асинхронные действия на которые можно запустить подписку

RxJS оптимизирует выполнение асинхронных действий. Если на асинхронное действие не добавлено ни одной подписки, это значит, что асинхронное действие не выполняется. Поэтому выполнение действия откалывается до появления первой подписки


# Провайдеры в Angular и регистрации сервисов
- Ангуляр — минималистичный фреймворк. На уровне кода он предоставляет много различных сервисов, но чтобы ими воспользоваться в конкретном приложении эти сервисы нужно подключить самостоятельно. Эти подключения осуществляются в файле `app.config.ts`


# Http запросы в Angular

- Все вспомогательные действия в Angular делаются с помощью сервисов
Angular имеет встроенный в фреймворк сервис для отправки запросов (он называется `HttpClient`). Этот сервис там где он нужен вставляется с помощью Dependency Injection. В основе этого сервиса лежит библиотека `RxJS`

- Чтобы сервис заработал его нужно подключить в `app.config.ts` (httpClientProvider()) в списке провайдеров

- Начиная с 21 версии Angular Http Client подключен во все приложения по умолчанию и его явно прописывать в конфиге необязательно
